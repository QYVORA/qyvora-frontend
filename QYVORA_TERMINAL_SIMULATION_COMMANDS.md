## Shell Features

### Prompt Format

```
kali@kali:~$
```

When running as root:

```
kali@kali:~#
```

When inside `msfconsole`:

```
msf6 >
```

### Variable Expansion

- `$VARIABLE` — expanded from the environment
- `$?` — expanded to the last exit code

### History Expansion

| Pattern | Behaviour |
|---|---|
| `!!` | Repeats the last command |
| `!n` | Repeats history entry `n` (1-indexed) |

### Glob Expansion

Patterns `*`, `?`, and `[...]` are expanded against the current directory in the VFS.

- `*.txt` — matches all `.txt` files in the current directory
- `?.*` — matches single-character filenames with an extension
- Hidden files (starting with `.`) are excluded from glob expansion

### I/O Redirection

| Syntax | Behaviour |
|---|---|
| `cmd > file` | Redirect stdout to file (overwrite) |
| `cmd >> file` | Redirect stdout to file (append) |
| `cmd < file` | Redirect stdin from file |

### Pipes

| Syntax | Behaviour |
|---|---|
| `cmd1 \| cmd2` | Pipe stdout of `cmd1` to stdin of `cmd2` |

Multi-stage pipes are supported: `cmd1 | cmd2 | cmd3`.

### Logical Operators

| Syntax | Behaviour |
|---|---|
| `cmd1 && cmd2` | Run `cmd2` only if `cmd1` succeeds (exit 0) |
| `cmd1 \|\| cmd2` | Run `cmd2` only if `cmd1` fails (exit != 0) |
| `cmd1 ; cmd2` | Run both regardless of exit code |

### Semicolon Separator

Commands separated by `;` execute sequentially regardless of exit code.

### Keyboard Shortcuts

| Shortcut | Action |
|---|---|
| `Ctrl+A` | Jump to start of line |
| `Ctrl+E` | Jump to end of line |
| `Ctrl+U` | Delete to start of line |
| `Ctrl+K` | Delete to end of line |
| `Ctrl+W` | Delete word before cursor |
| `Ctrl+L` | Clear screen |
| `Ctrl+C` | Cancel current input |
| `Ctrl+D` | Exit (on empty line) |
| `Ctrl+R` | Reverse-search history |
| `Tab` | Auto-complete files/paths |
| `Tab Tab` | Show all completions |
| `Up/Down` | Navigate command history |

## Navigation Commands

### `ls`

**Purpose:** List directory contents.

**Syntax:** `ls [OPTIONS] [PATH]`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-a` | Show hidden files (entries starting with `.`) |
| `-l` | Long listing format (permissions, owner, group, size, name) |
| `-d` | List directories themselves, not their contents |

**Behaviour:**
- Without flags: lists non-hidden entries, appending `/` to directories and `*` to executables
- `-a`: includes `.` and `..` plus hidden files
- `-l`: shows `total` line, permissions, owner, group, size, and name with `*` suffix for executables
- Targeting a file (not a directory): returns just the filename
- Non-existent path: returns error exit code 2

**Examples:**
```
ls
ls -la
ls -a /etc
ls -ld /home/kali/Desktop
```

---

### `cd`

**Purpose:** Change current working directory.

**Syntax:** `cd [DIR]`

**Supported Arguments:**

| Argument | Description |
|---|---|
| `DIR` | Target directory path (absolute or relative) |
| `-` | Return to previous directory (via `$OLDPWD`) |
| *(none)* | Go to `$HOME` |

**Behaviour:**
- Sets `$OLDPWD` to the previous `$cwd`
- Supports `~` and `~/path` expansion
- Supports `..` parent navigation
- Error if path does not exist or is not a directory

**Examples:**
```
cd /etc
cd ..
cd ~
cd -
cd Documents
```

---

### `pwd`

**Purpose:** Print current working directory.

**Syntax:** `pwd`

**Behaviour:** Returns the value of `$cwd`. No arguments supported.

---

### `tree`

**Purpose:** Display directory tree.

**Syntax:** `tree [PATH]`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-a` | Show hidden files |

**Behaviour:**
- Renders a tree with `├──`, `└──`, and `│` connectors
- Directories sorted before files, then alphabetically
- Appends `/` to directory names
- Shows total directory and file count at the end

**Examples:**
```
tree
tree /home/kali
tree -a /etc
```

## File Operation Commands

### `cat`

**Purpose:** Display file contents.

**Syntax:** `cat [FILE]...`

**Behaviour:**
- Concatenates multiple files with newline separator
- Empty output if no file given
- Error if file is a directory or does not exist

**Examples:**
```
cat welcome.txt
cat /etc/hostname /etc/hosts
```

---

### `echo`

**Purpose:** Print text to stdout.

**Syntax:** `echo [TEXT]...`

**Behaviour:**
- Joins all arguments with spaces
- Expands `$VARIABLE` references
- Empty output if no arguments

**Examples:**
```
echo Hello, World!
echo $HOME
echo "User: $USER"
```

---

### `touch`

**Purpose:** Create an empty file.

**Syntax:** `touch [FILE]...`

**Behaviour:**
- Creates new file with empty content, permissions `-rw-r--r--`
- Does nothing if file already exists
- Error if no file operand given

**Examples:**
```
touch newfile.txt
touch /tmp/a.txt /tmp/b.txt
```

---

### `mkdir`

**Purpose:** Create a directory.

**Syntax:** `mkdir [DIR]...`

**Behaviour:**
- Creates directory with permissions `drwxr-xr-x`
- Error if directory already exists
- Error if no operand given

**Examples:**
```
mkdir newdir
mkdir -p /tmp/a/b/c
```

Note: `-p` is not implemented; nested directory creation is not supported.

---

### `rm`

**Purpose:** Remove files or directories.

**Syntax:** `rm [OPTIONS] [FILE]...`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-r` | Recursive (required for directories) |
| `-f` | Force (suppress "No such file" errors) |
| `-rf` | Combined recursive + force |

**Behaviour:**
- Without `-r`: error if target is a directory
- With `-r`: removes directory and all children
- With `-f`: ignores missing files silently
- Targets are extracted by filtering out flags

**Examples:**
```
rm file.txt
rm -r old_directory
rm -rf unwanted_folder
rm -f nonexistent.txt
```

---

### `cp`

**Purpose:** Copy files or directories.

**Syntax:** `cp [OPTIONS] SRC DEST`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-r` | Recursive copy (required for directories) |
| `-rf` | Combined recursive + force |

**Behaviour:**
- Requires at least 2 non-flag arguments (source and destination)
- Creates a shallow copy of the source node at the destination path
- Error if source does not exist, is a directory without `-r`, or destination parent does not exist

**Examples:**
```
cp file.txt backup.txt
cp -r dir1 dir2
```

---

### `mv`

**Purpose:** Move or rename files.

**Syntax:** `mv SRC DEST`

**Behaviour:**
- Requires exactly 2 non-flag arguments
- Removes source from parent and adds to destination parent
- Renames if destination is a directory path
- Error if source or destination parent does not exist

**Examples:**
```
mv old.txt new.txt
mv file.txt /tmp/
mv dir1 /tmp/dir2
```

---

### `chmod`

**Purpose:** Change file permissions.

**Syntax:** `chmod MODE FILE`

**Supported Modes:**

| Mode | Behaviour |
|---|---|
| `+x` | Sets executable bits (`x`) for all users |
| `a+x` | Same as `+x` |

**Behaviour:**
- Only `+x` and `a+x` are implemented
- Other octal or symbolic modes are silently ignored

**Examples:**
```
chmod +x script.sh
chmod a+x executable
```

---

### `head`

**Purpose:** Show first lines of a file.

**Syntax:** `head [-n NUM] FILE`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-n NUM` | Show first `NUM` lines (default: 10) |

**Examples:**
```
head file.txt
head -n 5 file.txt
```

---

### `tail`

**Purpose:** Show last lines of a file.

**Syntax:** `tail [-n NUM] FILE`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-n NUM` | Show last `NUM` lines (default: 10) |

**Examples:**
```
tail file.txt
tail -n 20 file.txt
```

---

### `wc`

**Purpose:** Count lines, words, and characters.

**Syntax:** `wc FILE`

**Behaviour:** Returns `lines words chars filename`. No flags are implemented; always returns all three counts.

**Examples:**
```
wc file.txt
wc /etc/passwd
```

---

### `grep`

**Purpose:** Search file contents.

**Syntax:** `grep [OPTIONS] PATTERN FILE`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-i` | Case-insensitive matching |
| `--ignore-case` | Same as `-i` |
| `--color=auto` | (accepted but color output not rendered) |
| `--color=never` | (accepted, no effect) |

**Behaviour:**
- Plain substring matching (not regex) for simple patterns
- Regex support for patterns starting with `^`
- Returns matching lines, exit code 1 if no matches

**Examples:**
```
grep password /etc/shadow
grep -i root /etc/passwd
grep "^root" /etc/passwd
```

---

### `find`

**Purpose:** Search for files in a directory tree.

**Syntax:** `find [PATH] [-name PATTERN] [-type TYPE]`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-name PATTERN` | Filter by name (supports `*` glob in pattern) |
| `-type TYPE` | Filter by type: `file` or `dir` |

**Behaviour:**
- Recursively walks the VFS from the starting path
- Matches are absolute paths
- Without flags, returns all files and directories

**Examples:**
```
find /home/kali
find . -name "*.txt"
find / -type f
find . -name ".*"
```

---

### `sort`

**Purpose:** Sort file contents line by line.

**Syntax:** `sort [-r] FILE`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-r` | Reverse sort order |

**Behaviour:** Reads file, splits by newlines, sorts alphabetically (or reverse), returns sorted output.

**Examples:**
```
sort names.txt
sort -r names.txt
```

---

### `less` / `more`

**Purpose:** View file contents with paging indication.

**Syntax:** `less FILE` / `more FILE`

**Behaviour:**
- Shows first 40 lines of the file
- Appends `-- More --(N%)` indicator
- `less` and `more` are identical (more is an alias of less)
- No interactive scrolling (simulated only)

**Examples:**
```
less /etc/passwd
more file.txt
```

---

### `diff`

**Purpose:** Compare two files line by line.

**Syntax:** `diff FILE1 FILE2`

**Behaviour:**
- Shows differences in unified-like format with line numbers
- Capped at 20 differences displayed
- Exit code 0 if identical, 1 if different

**Examples:**
```
diff file1.txt file2.txt
```

---

### `ln`

**Purpose:** Create links.

**Syntax:** `ln [-s] SRC DEST`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-s` | Symbolic link (accepted, no VFS effect) |

**Behaviour:**
- Hard links: validates source exists, returns success
- Symbolic links (`-s`): returns success without actual VFS modification
- Both modes are stub implementations

**Examples:**
```
ln file.txt link.txt
ln -s file.txt symlink.txt
```

---

### `du`

**Purpose:** Estimate file space usage.

**Syntax:** `du [-h] [PATH]`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-h` | Human-readable sizes (e.g., `1.2K`) |

**Behaviour:**
- Walks the directory tree recursively
- Shows size per file/directory
- Shows total at the end
- Size is content length for files, sum of children for directories

**Examples:**
```
du .
du -h /home/kali
```

---

### `df`

**Behaviour:** Returns a static fake output showing three filesystems. No flags are processed.

**Output (example):**
```

tmpfs            2032764        0   2032764   0% /dev/shm
```

### `tar`

**Purpose:** Archive utility.

**Syntax:** `tar -c|-x|-t [-f ARCHIVE] [FILES...]`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-c` / `--create` | Create archive |
| `-x` / `--extract` | Extract archive |
| `-t` / `--list` | List archive contents |
| `-f NAME` / `--file NAME` | Archive filename |

**Behaviour:**
- `-c`: returns confirmation message listing files
- `-x`: returns extraction confirmation
- `-t`: returns a hardcoded example file listing (`README.md`, `src/`, `src/index.js`, etc.)
- Error if none of `-c`, `-x`, or `-t` provided

**Examples:**
```
tar -cf archive.tar file1 file2
tar -xf archive.tar
tar -tf archive.tar
```

---

### `zip`

**Purpose:** Compress files into a zip archive.

**Syntax:** `zip ARCHIVE FILE...`

**Behaviour:** Returns confirmation message. Does not actually modify the VFS.

**Examples:**
```
zip backup.zip file1.txt file2.txt
```

---

### `unzip`

**Purpose:** Extract a zip archive.

**Syntax:** `unzip FILE`

**Behaviour:** Returns hardcoded extraction output. Does not actually modify the VFS.

**Examples:**
```
unzip archive.zip
```

---

### `xxd`

**Purpose:** Hex dump of a file.

**Syntax:** `xxd FILE`

**Behaviour:**
- Reads file content and produces hex + ASCII representation
- Groups bytes in 16-byte chunks per line
- Format: `offset: hex_bytes  ascii`

**Examples:**
```
xxd /etc/hostname
```

---

### `strings`

**Purpose:** Extract printable strings from a file.

**Syntax:** `strings FILE`

**Behaviour:**
- Matches sequences of 4+ alphanumeric/hyphen/dot/slash characters
- Returns each string on its own line

**Examples:**
```
strings /etc/hostname
```

---

### `file`

**Purpose:** Determine file type.

**Syntax:** `file FILE`

**Behaviour:**
- Determines type based on file extension
- Supported types: txt, md, html, js, ts, py, json, xml, csv, png, jpg, gif, pdf, zip, tar, gz, sh, bin
- Directories return `directory`
- Unknown extensions return `data`

**Examples:**
```
file welcome.txt
file /etc/hostname
```

---

### `md5sum`

**Purpose:** Compute MD5 hash of a file.

**Syntax:** `md5sum FILE`

**Behaviour:**
- Returns a 32-character hex hash (simulated, not cryptographically correct)
- Hash is derived from filename + content using a deterministic function

**Examples:**
```
md5sum welcome.txt
```

---

### `sha256sum`

**Purpose:** Compute SHA-256 hash of a file.

**Syntax:** `sha256sum FILE`

**Behaviour:**
- Returns a 64-character hex hash (simulated, not cryptographically correct)
- Hash is derived from the same `fakeHash` function, doubled and truncated

**Examples:**
```
sha256sum welcome.txt
```

---

### `awk`

**Purpose:** Text processing.

**Syntax:** `awk '{print $N}' [FILE]`

**Supported Patterns:**

| Pattern | Description |
|---|---|
| `{print $0}` | Print entire line |
| `{print $N}` | Print Nth field (1-indexed) |
| `{print}` | Same as `{print $0}` |

**Behaviour:**
- Supports stdin or file input
- Splits fields on whitespace
- Only `{print ...}` patterns are supported (not full awk scripting)

**Examples:**
```
awk '{print $1}' /etc/passwd
cat /etc/passwd | awk '{print $3}'
```

---

### `sed`

**Purpose:** Stream editor.

**Syntax:** `sed 's/OLD/NEW/[g]' [FILE]`

**Supported Expressions:**

| Expression | Description |
|---|---|
| `s/old/new/` | Replace first occurrence per line |
| `s/old/new/g` | Replace all occurrences per line |

**Behaviour:**
- Only `s/.../.../` substitution expressions supported
- Supports stdin or file input

**Examples:**
```
sed 's/kali/student/' /etc/hostname
cat file.txt | sed 's/foo/bar/g'
```

---

### `cut`

**Purpose:** Extract columns from text.

**Syntax:** `cut -f N [-d DELIM] [FILE]`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-f N` | Field number(s), comma-separated (1-indexed) |
| `-d DELIM` | Field delimiter (default: tab) |

**Behaviour:**
- Supports stdin or file input
- Multiple fields: `-f 1,3` returns fields 1 and 3

**Examples:**
```
cut -f1 -d: /etc/passwd
cat data.csv | cut -f2 -d,
```

---

### `uniq`

**Purpose:** Remove duplicate adjacent lines.

**Syntax:** `uniq [FILE]`

**Behaviour:**
- Removes consecutive duplicate lines
- Supports stdin or file input
- Does not sort (unlike `sort | uniq`)

**Examples:**
```
uniq sorted.txt
sort names.txt | uniq
```

---

### `tr`

**Purpose:** Translate or delete characters.

**Syntax:** `tr SET1 SET2` or `tr -d SET1`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-d` | Delete characters in SET1 |

**Behaviour:**
- Without `-d`: character-by-character translation from SET1 to SET2
- With `-d`: removes all characters in SET1 from stdin
- Requires stdin input

**Examples:**
```
echo "hello" | tr a-z A-Z
echo "hello" | tr -d l
```

### `ps`

**Purpose:** List processes.

**Syntax:** `ps [OPTIONS]`

**Supported Flags:**

| Flag | Description |
|---|---|
| `-a` | Show all processes (not just current user) |
| `aux` | BSD-style full listing |
| `-ef` | System V-style full listing |

**Behaviour:**
- Without flags: shows processes for current user only (PID, TTY, TIME, CMD)
- With `-a`, `aux`, or `-ef`: shows all processes with full details (USER, PID, %CPU, %MEM, VSZ, RSS, TTY, STAT, START, TIME, COMMAND)
- Returns hardcoded process list (init, systemd-journald, sshd, bash, etc.)

**Examples:**
```
ps

ps -ef
ps -a
```

---

### `top`

**Purpose:** Show process overview.

**Syntax:** `top`

**Behaviour:** Returns a single snapshot (not interactive) with random load averages, task counts, CPU usage, and memory stats.
